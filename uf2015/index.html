About Security

3 Phases of Security

  1. Protect the site with some simple services and tools.
  2. Detect when a problem has occurred.
  3. Easily Recover when something goes wrong.

Vulnerabilities

The OWASP Top 10 - A List of the top 10 vulnerabilities affecting all applications.

WordPress Core, by design, can handle some of these for all sites.

Poorly vetted plugins and themes and server misconfigurations are the source of most application problems in WordPress.

Hacking is Automated

"Bots" will get you - not individuals

  1. They don't need a target
  2. Many sites in little time
  3. Often "dumb" (but getting smarter)

Rarely is the average WordPress site targeted directly

See what you can learn from your site: http://wpscan.org

How Do They Get In?

  • Sometimes they don't need to get in at all (Denial of Service)
  • Through the application itself (typically a plugin or theme)
    • Exploit a vulnerable file
    • Keep guessing your password (brute force)
    • Intercept your credentials (man in the middle)
  • Through the server
    • Keep guessing your password (brute force)
    • Infect another user
    • Intercept your password (man in the middle)
    • Exploit vulnerable server software

Protecting Your Site

Update, Update, Update

One of the best aspects of WordPress security: there are so many eyes on it.

When something is found the vulnerabilities are often quickly patched and free to install.

Follow Best Practices (admins)

  • Enforce strong and rotating passwords
  • Enforce SSL (even on a closed network)
  • Limit admin accounts
  • Use trusted plugins (stick with large, established plugins when possible)
  • Remove extras (plugins, themes, users, etc)
  • Use appropriate moderation

Follow Best Practices (server)

  • Use a valid SSL certificate
  • Don't use FTP
  • Create regular backups
  • Use firewalls
  • Use Fail2ban

Follow Best Practices (development)

  • Use Version Control
  • Sanitize, escape and validate
  • Always perform code reviews
  • Use WordPress APIs
